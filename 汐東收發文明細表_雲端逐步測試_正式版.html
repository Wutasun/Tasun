汐東收發文明細表｜雲端逐步測試＋Console＋正式報告

本頁整合 R506～R517 雲端逐步測試正式版，固定檔名為「汐東收發文明細表_雲端逐步測試_正式版.html」，且 R506 起可不經 index.html 直接開啟測試工具畫面，供 GitHub Pages 後續固定覆蓋更新。測試會同時檢查「汐東收發文明細表.html、汐東收發文登錄表.html、汐東回覆文登錄表.html」三頁之間的連結、共用 resourceKey/dbName/tableId、回覆來源文號寫回同列，以及雲端 Read/Merge/再 Read 同步流程。

R516 測試判定原則（測試頁免 index 直接開啟）

  • 正式頁版本只讀 tasun-version.json 中本頁 page entry，不得因 root version 或其他頁版本造成互相導向。
  • 本測試頁可直接開啟，不導回 index.html；但雲端 Read/Merge 測試仍會依登入 session bridge 與 Worker 權限判定。
  • 同步狀態必須以雲端流程為依據：Health 可到達 → Read 成功 → admin/write Merge 成功 → 再 Read 校正成功 → 正式頁 UI 顯示已同步。
  • 不得開頁即直接顯示「已同步」；若未完成雲端確認，必須顯示同步處理中、待登入授權、連線失敗、同步失敗或等待同步。
  • 三個收發文頁面必須共用 official-documents / tasun_official_documents_idb_v1 / officialDocumentTable，且登錄表、回覆文登錄表需能回寫同一份明細資料。
  • 報告會列出錯誤問題與建議修復方向，供後續依報告修正。
  • R512 版本刷新規則：正式頁即使網址仍帶 r510/r511 舊 v 參數，也必須改用本頁 page entry 與 embedded 版本取較新者；同時清除舊版版本鎖、Cache Storage、Service Worker 與舊 build stamp memo，不得被舊版覆蓋。
  • R514 表頭科技色彩規則：「回覆文號(監造)、回覆日期(監造)、處理情形、展延日期、展延距今日天數」需有與一般欄位不同的亮麗科技感表頭顏色，並納入自我修復檢查。
  • R515 監造期限欄位規則：「處理期限(監造)」需改為「處理期限(日)(監造)」，「期限日期(監造)」需改為「距今日天數(監造)」，且距今日天數(監造)欄資料文字需為粗體螢光科技色。
  • R516 收發文登錄表規則：不顯示筆數與登錄說明文字；登錄表只顯示項次至到期日期(監造)等指定 20 欄，且輸入框需左右 0.1cm 內距、文字由左而右、自動換行與自動增高。
  • R517 登錄表表單檢查：收發文登錄表全部指定欄位需在表單內；除主旨外輸入框採緊湊尺寸；項次需依明細資料最大項次 +1 自動顯示，無資料時顯示 1，且項次輸入框寬度以 6 位數為基準。
Merge 測試使用 deleted:true 的 probe tombstone，不會新增正式顯示資料。若目前角色不是 admin/write，Merge 會列為未授權並顯示修復建議。
