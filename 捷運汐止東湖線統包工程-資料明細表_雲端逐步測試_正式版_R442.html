用途：一步一步測試雲端 Health / Read / Merge、正式頁版本、Tasun v5 固定口令、自我修復紀錄與 Console 錯誤，最後自動產生 TXT / JSON 報告；報告會列出錯誤問題與建議修復方向，供後續依報告修復。

正式測試判定原則

  • 正式版號唯一來源必須為 tasun-version.json / tasun-version-loader.js，測試會比對是否仍殘留 R419 或頁面顯示非 R442。
  • 正式頁必須符合 Tasun v5 全站固定口令總版＋TasunSelfHealV5＋R268 統一自我修復標準；測試會檢查 pageKey、resourceKey、dbName、build stamp、自我修復清單與雲端權威同步規則。
  • 雲端狀態不得長時間停在「同步檢查中」；若 iframe 實際頁面檢查仍顯示同步檢查中，報告會列為需修復。
  • Console error / warn / unhandledrejection 會納入正式報告。
Merge 測試使用 deleted:true 的 probe tombstone，不會新增正式顯示資料。
